Apertura e Benvenuto

Introduzione dell'evento e dei relatori

09:45 - 10:30

Continuous Integration and Deployment Best Practices on AWS

di Danilo Poccia (Solutions Architect at Amazon Web Services, )

Lavorando con un'infrastruttura 100% API-driven è possibile sviluppare ed eseguire applicazioni con velocità e flessibilità come mai prima d'ora. Vi presentiamo alcuni concetti chiave e design pattern per Continuous Deployment and Continuous Integration, due elementi del Lean Development delle applicazioni e delle infrastrutture, per migliorare la produttività e l'innovazione con AWS.

10:30 - 11:15

Make sense of your (BIG) data!

di David Pilato (Technical Advocate at ElasticSearch, )

Elasticsearch is a search server based on Lucene. It provides a distributed, multitenant-capable full-text search engine with a RESTful web interface and schema-free JSON documents. You can use it as well to compute information on live data. Kibana is a full web application built with AngularJS. It's a generic and powerful visualisation tool for your data. Both projects are under Apache 2 Licence. During this session, you will discover how elasticsearch actually works. We will inject marketing data into elasticsearch and build live a dashboard using Kibana. In minutes, you will know how to build YOUR own dashboard and make sense of YOUR data.

11:15 - 11:45

Coffee break!

11:45 - 12:30

TRACCIA REST & FRONTEND

Problemi e soluzioni nello sviluppo di API RESTful

di Enrico Zimuel (Senior Software Engineer at Zend Technologies, )

Sviluppare (o consumare) API RESTful non è un compito banale, sono tanti gli aspetti tecnici da tener presente. Dalla gestione dalla negoziazione del contenuto(content negotiation), al versionamento (versioning), dalla gestione degli errori alla documentazione, passando per l'autenticazione, la gestione dei permessi, etc. In questo talk verranno illustrate le caratteristiche principali da tenere presente durante lo sviluppo di API RESTful, presentando alcuni standard emergenti come HAL JSON e HTTP Problem e standard già affermati come OAuth2 (RFC 6749) per l'autenticazione. Per finire, verrà introdotto brevemente il progetto Apigility, un software open source per facilitare la creazione di API.

TRACCIA DEPLOY

Saltstack, Provisioning Systems And Scalable Deploy Strategies

di Luca Cipriani (Founder of Sysapparchitect, )

Il deploy è un punto cruciale nella gestione di ogni applicazione. In questo talk vedremo Saltstack come un sostituto asincrono di Puppet per il provisioning in un'architettura push / pull con più di 100 nodi. Scopriamo quanto è semplice implementare strategie per piccoli progetti con Fabric.

12:30 - 13:15

TRACCIA REST & FRONTEND

AngularJS & Cloud Resources

di Gabriele Mittica (co-founder of Corley srl, )

Grazie alle potenzialità del framework JavaScript di Google possiamo integrare al meglio risorse cloud in una Single Page Application: scopriamo come interagire con servizi di storage remoti senza dipendenze da backend, consumare API RESTful e interagire con sistemi di code e notification di Amazon Web Services grazie agli strumenti di AngularJS.

TRACCIA DEPLOY

Deployment and Management on AWS: A Deep Dive on Options and Tools

di Danilo Poccia (Solutions Architect at Amazon Web Services, )

Approfondimento tecnico su come gestire il ciclo di vita di intrastrutture ed applicationi usando servizi AWS quali Elastic Beanstalk, per un deploy rapido e la gestione delle applicazioni, OpsWorks, per modellare l'intera applicazione usando stack e layer, a CloudFormation, per descrivere risorse AWS e ogni dipendenza associata attraverso l'utilizzo di template.

13:15 - 14:15

Pranzo (offerto in loco)

14:15 - 15:00

TRACCIA REAL-TIME WEB

Build real-time applications with Javascript

di Antonello Pasella (Web Consultant at 1UP Academy, )

Nelle applicazioni multiutente spesso vige la logica del "Chi salva per ultimo sovrascrive tutto". Nel 2014 è ancora necessario? Aggiungiamo un pizzico di realtime al nostro applicativo con Firebase, che ci permette, attraverso API, di memorizzare e sincronizzare dati in millisecondi, e di sviluppare app mobile e web in pochi minuti con client-side coding.

TRACCIA DATABASE

Costruisci il TUO database con LevelDB e Node.js

di Matteo Collina (Code pirate, )

Avete mai pensato di scrivere un vostro database? Un Key/Value store, o un database documentale? Oppure un Graph DB? O magari volete un database che si sincronizzi fra Browser e Server in modo trasparente? Nodebase è il movimento che fa per voi! Un gruppo di mad scientist nella comunità Node.js hanno preso l'ultraveloce LevelDB e hanno sviluppato decine di piccole librerie che consentono di aggiungerci tutte le funzionalità di cui avete bisogno, dalla replicazione all'indicizzazione: stiamo parlando di LevelUp!

15:00 - 15:45

TRACCIA REAL-TIME WEB

Cloud-ready web messaging with CometD

di Simone Bordet (Lead Architect at Webtide e Project Lead at CometD, )

Il progetto open source CometD è un sistema di web messaging veloce e scalabile basato su standard JEE. Applicazioni come chat room, videogiochi online, sistemi di trading, e portali di notizie possono utilizzare CometD per implementare una soluzone facile da scrivere, veloce, scalabile e transport-independent. In questa sessione esploreremo le funzioni di CometD, quali il supporto ai WebSocket e el funzionalità cloud-oriented che rendono la tua applicazione CometD pronta a scalare.

TRACCIA DATABASE

Database: backup e disaster recovery in Cloud

di Walter Dal Mut (co-founder of Corley srl, )

Viaggio dal Backup & Restore fino al Multi-Site Disaster Recovery (on-premise e cloud), con una particolare attenzione ai database. Scopriamo le migliori strategie e metodologie, attraverso esempi reali e case study legati a MySQL e MongoDB, con l'obiettivo di garantire la continuità di servizio grazie a soluzioni in alta disponibilità.

15:45 - 16:15

Coffee break!

16:15 - 17:00

Playlists at Spotify

di Jimmy Mårdell (Senior Developer at Spotify)

Playlists are a central part of Spotify. To date, our users have created over 1 billion playlists! This means a lot of data that needs to be stored and accessed on a regular basis. Users can subscribe to other users playlists, create and update playlists in offline mode and receive real-time updates when playlists are changed. This presentation will go into detail about some of the techncial challenges involved in the playlist system, and how they were solved. The system is written in Java using Apache Cassandra as the backend storage solution.

17:00 - 17:45

Da Redis a Redis Cluster

di Salvatore Sanfilippo (Lead Developer at Redis, )

In questo talk, Salvatore Sanfilippo, lead developer di Redis, racconta le differenze, sia dal punto di vista applicativo che da quello delle operazioni, che ci sono tra Redis eseguito su singola istanza, e Redis Cluster. Alcuni dei temi trattari riguardano l'architettura complessiva di Redis Cluster, le situazioni nelle quali vale la pena migrare al Cluster, e a quali compromessi deve scendere l'applicazione rispetto al passato. In fine saranno analizzati lo stato attuale del progetto dal punto di vista della maturita' e stabilita', e la roadmap per l'immediato futuro.
